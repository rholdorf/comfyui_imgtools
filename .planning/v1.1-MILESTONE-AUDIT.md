## Integration Issues (1 critical)

### FaceCropAlign drops pose key from crop_landmarks_out

**Location:** `face_crop.py` lines 85-88
**Root cause:** `crop_landmarks_out` dict only includes `landmarks` and `landmarks_3d` keys. The `pose` key from upstream `FaceDetect` face dicts is not forwarded.

**Impact:**
- POSE-04 (auto-attenuation) — never triggers through standard node chain
- MRPH-01 (pose-aware delta) — always falls back to Procrustes

**Why it wasn't caught earlier:** Phase 10 fixed FaceDetect to emit pose data, and integration tests verified the pose-aware path using synthetic face dicts. But the tests bypass FaceCropAlign — they feed pose data directly to FaceModelMorph.

**Fix:** One-line addition in `face_crop.py`:
```python
crop_landmarks_out = [{
    "landmarks": crop_landmarks,
    "landmarks_3d": landmarks[idx].get("landmarks_3d", np.zeros((478, 3))),
    "pose": landmarks[idx].get("pose", None),   # forward pose data
}]
```
