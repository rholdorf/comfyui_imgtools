```python
def _passthrough(source_image, h, w, align_data):

    return (source_image, full_mask, align_data)
```

  <behavior>
    **ComfyUI conventions (test first):**
    - INPUT_TYPES has required keys: original_image (IMAGE), morphed_face (IMAGE), warp_mask (MASK), align_data (ALIGN_DATA)
    - RETURN_TYPES is ("IMAGE", "MASK")
    - RETURN_NAMES is ("composited_image", "face_region_mask")
    - FUNCTION is "composite"
    - CATEGORY is "imgtools/face"

    **Passthrough / graceful degradation:**
    - Missing key in align_data -> returns (original_image, empty_mask)
    - Singular transform_matrix -> returns (original_image, empty_mask)
    - morphed_face dimensions != crop_box dimensions -> returns (original_image, empty_mask)
    - Empty/zero-size crop_box -> returns (original_image, empty_mask)
    - No crash on any input combination

    **Identity transform round-trip (no rotation):**
    - With identity transform_matrix and morphed_face == cropped region of original:
      composited face region should match original within RMSE < 0.001
    - Output dimensions match original_image dimensions exactly

    **Rotated face round-trip:**
    - With rotated transform (e.g. 15 degrees) and strength=0 morph:
      composited face region should be close to original within RMSE < 0.25 (double interpolation loss)

    **Alpha blending:**
    - Where mask == 1.0: result pixels == morphed face pixels (after reverse transform)
    - Where mask == 0.0: result pixels == original image pixels
    - Feathered mask region: result is weighted blend of both

    **face_region_mask output:**
    - Is the reverse-transformed warp_mask in original image space
    - Has same H, W as original_image
    - Values in [0.0, 1.0]

    **Crop region expansion:**
    - Canvas placement expands crop_box by small margin (5px) for interpolation safety
    - Expansion is clamped to image bounds
  </behavior>
