<task type="auto" tdd="true">
  <name>Task 1: Alignment utilities with TDD</name>
  <files>utils/alignment.py, tests/test_alignment.py, tests/conftest.py</files>
  <behavior>
    - compute_eye_centers: given (478, 2) landmarks array, returns (left_eye_center, right_eye_center) each as (x, y) mean of LEFT_EYE_INDICES / RIGHT_EYE_INDICES
    - compute_alignment_angle: given left_eye=(100,100) right_eye=(200,100) (horizontal), returns 0.0 radians; given left_eye=(100,110) right_eye=(200,100) (tilted ~-5.7 deg), returns negative angle
    - compute_alignment_angle: already-upright eyes (same y) returns 0.0
    - build_alignment_transform: returns (AffineTransform, float_angle) tuple; transform rotates around eye midpoint
    - apply_alignment: given (H,W,3) float array and transform, returns same-shape aligned array
    - compute_padded_crop_box: padding=0.0 gives tight bbox of landmarks; padding=0.5 expands by 50% each side; result clamped to (0,0,img_w,img_h)
  </behavior>
  <action>
    First, add a `mock_multi_face_landmarks` fixture to tests/conftest.py: two face dicts with known landmark positions (not random -- use deterministic coords so eye centers and crop boxes are predictable). Also add a `mock_landmarks_tilted` fixture where right eye is higher than left eye (about 15 deg tilt).

    Then write failing tests in tests/test_alignment.py:
    - test_compute_eye_centers: verify left/right eye center is mean of correct indices
    - test_alignment_angle_horizontal: eyes at same y -> angle ~0
    - test_alignment_angle_tilted: eyes with known dy/dx -> expected angle via arctan2
    - test_zero_angle_no_rotation: build_alignment_transform with horizontal eyes, angle component is ~0
    - test_padded_crop_box_tight: padding=0 gives bounding box of landmarks
    - test_padded_crop_box_expanded: padding=0.3 expands bbox by 30%
    - test_padded_crop_box_clamped: landmarks near edge with large padding, result clamped to image bounds
    - test_apply_alignment_shape: output shape matches input shape

    Then implement utils/alignment.py:
    - LEFT_EYE_INDICES and RIGHT_EYE_INDICES constants (from research: 16 indices each)
    - compute_eye_centers(landmarks_px): mean of landmarks_px[LEFT_EYE_INDICES] and [RIGHT_EYE_INDICES]
    - compute_alignment_angle(left_eye, right_eye): np.arctan2(dy, dx)
    - build_alignment_transform(landmarks_px, img_width, img_height): builds AffineTransform rotating by -angle around eye midpoint. Returns (transform, angle). Use skimage AffineTransform composition: translate to origin, rotate, translate back.
    - apply_alignment(img_np, transform, output_shape=None): uses skimage.transform.warp with inverse_map=transform.inverse, order=1, mode='constant', preserve_range=True. Input/output are float64 (H,W,3).
    - compute_padded_crop_box(landmarks_px, transform, padding_factor, img_w, img_h): transform landmarks, get bbox, expand by padding_factor on each side, clamp to image bounds. Returns (x1, y1, x2, y2) as ints.

    IMPORTANT: Do NOT use OpenCV. Use skimage.transform.AffineTransform and skimage.transform.warp only (per PLAT-02).
    IMPORTANT: landmarks are (x, y) pixel coords. polygon2mask and image arrays use (row, col) = (y, x). Be explicit about conversions.
  </action>
  <verify>
    <automated>python -m pytest tests/test_alignment.py -x -v</automated>
  </verify>
  <done>All alignment tests pass: eye center computation, angle calculation (horizontal and tilted), crop box with padding (tight, expanded, clamped), and alignment application</done>
</task>
