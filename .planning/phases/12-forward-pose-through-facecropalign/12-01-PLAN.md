<objective>
Fix FaceCropAlign to forward the `pose` key from upstream face dicts into its `crop_landmarks_out` output, closing the last gap in the pose-aware morphing pipeline.

Purpose: Without this fix, FaceModelMorph always falls back to the Procrustes path when used through the standard node chain (FaceDetect -> FaceCropAlign -> FaceModelMorph), because FaceCropAlign strips the pose key. This is the final plumbing fix for v1.1 pose-aware morphing.

Output: One-line fix in face_crop.py + new pose forwarding tests in test_face_crop.py.
</objective>
